# Getting started
Вся реализация RAG-решения находится в ноутбуке gnn_contest.ipynb. Файл для контекста RAG'а и список вопросов подгружаются отдельно, в ту же директорию что и ноутбук.

Ответы на искомые в хакатоне вопросы лежат в файле answers.xlsx

# Описание решения

1. Построение графа
   - Узлы: Текстовые чанки.
   - Ребра:
       - Последовательные: Связи между чанком i и i+1
       - Семантические: Связи между чанками с высокой косинусной близостью (KNN), чтобы соединить определения и примеры, находящиеся в разных частях текста.
2. Использование GNN (GCN):
   - Используем GCN (Graph Convolutional Network) для распространения информации (Message Passing).
   - Вместо сложного обучения (которое требует размеченных данных, недоступных здесь), применяем Feature Propagation. Эмбеддинг каждого чанка обновляется, агрегируя информацию от (1) своих семантических соседей и (2) контекстных соседей.
   - Это позволяет находить чанки, которые сами по себе могут не иметь прямого лексического совпадения с вопросом, но связаны с темой через граф.
4. Этап улучшения: GNN улучшает эмбединги перед этапом поиска. Поиск происходит по "обогащенным" векторам.
